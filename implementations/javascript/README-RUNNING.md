# P2P Decentralized Emulation System - Готовая к использованию версия

Эта версия системы готова к немедленному запуску и использованию "под ключ".

## Быстрый старт

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Запустите систему:
   ```bash
   npm start
   ```

   Или для разработки:
   ```bash
   npm run dev
   ```

## Что происходит при запуске

При запуске системы:

1. Создается узел P2P с уникальным идентификатором DID
2. Инициализируется транспортный слой на базе libp2p
3. Узел начинает автоматически обнаруживать и подключаться к другим узлам
4. Создаются демонстрационные CRDT для синхронизации состояния
5. Состояние узла автоматически синхронизируется каждые 10 секунд
6. Узел обеспечивает интероперабельность с ActivityPub, Nostr и IPFS

## Взаимодействие с системой

### REST API (будет доступен на localhost:3000)
- `GET /state` - Получить текущее состояние узла
- `POST /state` - Обновить состояние узла
- `GET /peers` - Получить список подключенных узлов

### PubSub топики
- `p2p-state-sync` - Для синхронизации состояния между узлами
- `p2p-identity` - Для обмена идентификационной информацией

## Остановка системы

Для остановки системы нажмите `Ctrl+C`. Система корректно завершит все соединения и остановит узел.

## Расширение функциональности

Вы можете расширить функциональность системы, добавляя новые CRDT, обработчики сообщений и модули в следующие файлы:

- `src/StateSyncLayer.js` - Для добавления новых типов CRDT
- `src/TransportLayer.js` - Для добавления новых обработчиков сообщений
- `src/SemanticAdapter.js` - Для добавления поддержки новых протоколов
