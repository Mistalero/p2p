# Glossary

This glossary defines key terms and concepts used throughout the P2P Node documentation.

## A

### API (Application Programming Interface)
A set of protocols and tools for building software applications. In the context of P2P Node, APIs allow components and external applications to interact with the node.

### Asynchronous Processing
A method of processing where operations can be executed without blocking other operations. In P2P Node, asynchronous processing allows components to handle multiple events concurrently.

## C

### Component
A modular, reusable piece of software that implements specific functionality within the P2P Node system. Components are designed to be language-agnostic and can communicate with each other through standardized interfaces.

### Component Lifecycle
The sequence of stages that a component goes through during its existence: Initialization, Startup, Operation, and Shutdown.

### Content Negotiation
The process of selecting an appropriate data format for communication between components or nodes based on their capabilities and preferences.

## D

### Decentralized Network
A network architecture where control and decision-making are distributed among multiple nodes rather than being concentrated in a single authority.

### DID (Decentralized Identifier)
A new type of identifier that enables verifiable, decentralized digital identity. In P2P Node, DIDs are used for node identification.

## E

### Event
A data structure representing a change in the system state. Events in P2P Node contain a timestamp, author, action type, and payload.

### Event Log
A chronological record of events that have occurred in the system.

## F

### Format Versioning
A mechanism for managing different versions of data formats to ensure compatibility between components and nodes.

## G

### Glossary
A list of terms and their definitions, such as this document.

## H

### Hash
A fixed-size string of characters that uniquely represents data of any size. In P2P Node, hashes are used to ensure data integrity.

## I

### Immutable Data
Data that cannot be changed after it is created. In P2P Node, events are immutable to ensure consistency and traceability.

## J

### JSON (JavaScript Object Notation)
A lightweight data interchange format that is easy for humans to read and write and easy for machines to parse and generate.

## L

### Language Agnostic
The ability of a system or component to work with multiple programming languages without being tied to any specific one.

### Layered Architecture
An architectural pattern where the system is organized into layers, with each layer having a specific responsibility and interacting only with adjacent layers.

## M

### Message Envelope
A wrapper that contains both the payload data and metadata such as schema identifier, version, and integrity signature.

## N

### Node
An individual participant in the P2P network that can store data, process events, and communicate with other nodes.

## P

### Peer-to-Peer (P2P)
A network architecture where each participant (peer) can act as both client and server, sharing resources directly with other peers.

### Protocol
A set of rules and conventions that govern how data is transmitted and processed in a network.

## Q

### QUIC
A modern transport layer protocol that provides secure, multiplexed connections. Used as one of the transport protocols in P2P Node.

## R

### Replication
The process of copying and distributing data across multiple nodes in the network to ensure availability and resilience.

## S

### Serialization
The process of converting an object or data structure into a format that can be stored or transmitted and then reconstructed later.

### Schema
A definition of the structure and format of data, used to ensure consistency and validation.

## T

### TCP (Transmission Control Protocol)
A core protocol of the Internet protocol suite that provides reliable, ordered, and error-checked delivery of data.

### Transport Layer
The layer in the system architecture responsible for delivering messages between components and nodes.

## U

### URI (Uniform Resource Identifier)
A string of characters that identifies a resource. In P2P Node, URIs are used to identify nodes and components.

## W

### WebSocket
A protocol that provides full-duplex communication channels over a single TCP connection. Used as one of the transport protocols in P2P Node.

## Y

### YAML (YAML Ain't Markup Language)
A human-readable data serialization standard that can be used with all programming languages.

## Z

### Zero-Copy
A technique where data is transferred between layers without being copied, improving performance by reducing memory usage and processing time.
